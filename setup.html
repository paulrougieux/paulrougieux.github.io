This is a description of how I edit text files for persons looking at my screen and wondering what editor I use to move so fast. Does a text editor still matter in the age of AI?

Let me describe my text editing setup. This text was written by a human with his fingers on a keyboard in Vim.

Text editing

I use Vim or Neovim to edit markdown files and source code. Mostly python code as of 2025, occasionally R code or other programming languages as well.

When editing python code, the most used shortcut is <leaderkey>p which sends a paragraph to the ipython read–eval–print loop (REPL). This shortcut uses vim-slime to “grab some text and send it to a target” tmux pane.

Server setup

Manual setup of a modelling server

A server setup to run mostly python models.

Connect with alias in ssh config

See config in ~/.ssh/config The password can be saved in a storage file.

Create users

On the server

Log in as root and run these:

Create user:

adduser modeluser

Grant sudo:

usermod -aG sudo modeluser

Set password:

passwd modeluser.

Disable root SSH: Edit /etc/ssh/sshd_config, set PermitRootLogin no, then systemctl restart ssh.

On the laptop

Update your ~/.ssh/config alias to use User modeluser.

Copy the public key in remote server as authorized keys First check which public key is the most recent.

scp $HOME/.ssh/id_rsa.pub user@server:~/.ssh/authorized_keys

On the server

  • disable password authentication

    vi /etc/ssh/sshd_config

    PasswordAuthentication no ChallengeResponseAuthentication no PubkeyAuthentication yes

Setup python models

Create a virtual environment and install the model and its dependencies inside

cd ~
mkdir eu_cbm/.venv
python3 -m venv eu_cbm/.venv
source eu_cbm/.venv/bin/activate
cd ~/eu_cbm/eu_cbm_hat/
pip install -e .
pip install ipython # Inside the environment
pip install libcbm # Why is this not a dependency of eu_cbm_hat?

See ~/eu_cbm/eu_cbm_hat/docs/setup_on_jrc_bdap.md For AIDB initialization and model run instructions.

Synchronize to and from the server

rsync

Send laptop data to the server

rsync -avz ~/eu_cbm/TiMBA_Data/ paul@flower:eu_cbm/TiMBA_Data/

Send server data to the laptop

rsync -avz paul@flower:eu_cbm/cbm-timba-coupling/scripts ~/eu_cbm/cbm-timba-coupling/scripts

git with the server as a remote

Add the server as a remote

git remote add flower paul@flower:eu_cbm/cbm-timba-coupling

Fetch changes the server made

git fetch flower

See what’s different

git log HEAD..flower/main --oneline

Merge or cherry-pick as needed

git merge flower/main

Nix automated setup

My plan is to use nixos-infect to recreated my system on a Debian pre-created by the hosting provider.

Auto generated NixOS config for a VM

Declarative recreation of a Debian 13 server. These files describe the machine; getting NixOS onto the box is a separate, destructive step.

Files - configuration.nix — services, users, networking, packages. - hardware-configuration.nix — disk/boot scaffold. Regenerate on install with nixos-generate-config --root /mnt.

What was captured from the live Debian server

  • Host vmd...., BIOS/GRUB boot, ext4 root on /dev/sda1.
  • User paul (sudo via wheel), SSH key-only, no root login, no passwords.
  • Static IPv4 .... gw ..., IPv6 …:...........
  • Local mail (exim4 → opensmtpd), cron, timesyncd, resolved.
  • Tools: curl, git, htop, gdu, ipython3, ethtool.
  • Timezone Europe/Berlin, locale C.UTF-8.

Verify before trusting blindly - Interface name: NixOS may name the NIC ens3/enp1s0 rather than eth0. Check ip link and update systemd.network.networks (or match on MAC). - IPv6 gateway: set to fe80::1 as a guess — confirm with Netcup / ip -6 route. - hardware-configuration.nix: replace with the generated one.

Getting NixOS onto the machine (pick one) 1. nixos-anywhere (recommended, fully reproducible) — needs a disko disk spec; kexecs an installer over SSH and reinstalls from scratch. Wipes the disk. https://github.com/nix-community/nixos-anywhere 2. nixos-infect — converts the running Debian to NixOS in place. Faster, no console needed, but riskier. https://github.com/elitak/nixos-infect 3. Manual — boot Netcup rescue/NixOS ISO, partition, nixos-install.

After install, deploy changes with nixos-rebuild switch (or build remotely with nixos-rebuild switch --target-host root@212.90.121.218 --flake .#... if you migrate to a flake).
